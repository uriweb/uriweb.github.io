Usage

Create a tabbed interface with the class cl-tabs

<div class="cl-tabs"> <section> ... </section> <section> ... </section> ... </div>

Each <section> will become a panel and have an accompanying tab, which is generated dynamically using JavaScript. The non-JavaScript experience will not have tabs or hidden panels, and should be indistinguishable from any other page sections.

ID's and tab labels

An id attribute can be added for each section, but it is not required. If supplied, JavaScript will use the section id for the corresponding tab's href. Otherwise, it will create and use a generic sequential id in the format cl-tabs-section-#

JavaScript will try to use the section's headers as the label for the corresponding tab, beginning with <h1>. If an <h1> is absent, the first <h2> will be used. If both are absent, a generic sequential label will be created in the format "Section #" and used for the tab.

Accessibility

Because tabs, by design, hide information and contain related elements that are symantically detatched from one another (i.e. a tab and the content it controls are not sibling elements), role and aria-* attributes allow the component to be fully accessible. To make it easier to code, these attributes are added automatically through JavaScript.
