Tabs

Tabs allow a large amount of information to be presented in compact form. Tabs segregate content into sections and hide all but one section at a time, while allowing the user random access to each section.

  • Usage
  • Accessibility

Usage

Create a tabbed interface with the class cl-tabs

<div class="cl-tabs"> <section> ... </section> <section> ... </section> ... </div>

Each <section> will become a panel and have an accompanying tab, which is generated dynamically using JavaScript. The non-JavaScript experience will not have tabs or hidden panels, and should be indistinguishable from any other page sections.

Section IDs and Tab Labels

An id attribute can be added for each section, but it is not required. If supplied, JavaScript will use the section id for the corresponding tab's href. Otherwise, it will create and use a generic sequential id in the format cl-tabs-section-#

JavaScript will try to use the section's headers as the label for the corresponding tab, beginning with <h1>. If an <h1> is absent, the first <h2> will be used. If both are absent, a generic sequential label will be created in the format "Section #" and used for the tab.

Shortcode Support

Tabs require two shortcodes: one to create the tabs container, and one for each tab/section. The section shortcodes are self-closing and are nested inside the container shortcode, which is enclosing.

Tabs can be implemented with the shortcode cl-tabs

[cl-tabs][/cl-tabs]

Attributes

  • class (string) (optional)

    Set custom CSS class(es)

    (default value: none)

Add sections to the component with the shortcode cl-tab

[cl-tab title="A Great Tab"]Tabs are great - when they're accessible![/cl-tab]

The tab shortcode supports nested shortcodes, which makes adding other components easy.

[cl-tab title="An Excellent Tab"]Tabs are excellent. Here's a button! [cl-button ...][/cl-tab]

Attributes

  • title (string) (optional)

    The text for the tab and section title

    (default value: none)

  • id (string) (optional)

    Set custom ID for the section

    (default value: none)

  • class (string) (optional)

    Set custom CSS class(es)

    (default value: none)

Removed

  • body (bool) (optional)

    2.3.0Use an enclosing shortcode instead

Nest the cl-tab shortcode(s) inside the cl-tabs shortcode.

[cl-tabs][cl-tab title="Tab 1"]Tab 1 content.[/cl-tab][cl-tab title="Tab 2"]Tab 2 content.[/cl-tab]...[/cl-tabs]

Accessibility

Because tabs, by design, hide information and contain related elements that are symantically detatched from one another (i.e. a tab and the content it controls are not sibling elements), role and aria-* attributes allow the component to be fully accessible. To make it easier to code, these attributes are added automatically through JavaScript.

The final generated html looks like this:

<div class="cl-tabs cl-tabs-js"> <ul role="tablist"> <li role="presentation"> <a href="#cl-tab-section-1" id="cl-tabs-tab-1" role="tab" aria-selected="true">...</a> </li> <li role="presentation"> <a href="#cl-tab-section-2" id="cl-tabs-tab-2" role="tab" tabindex="-1">...</a> </li> <li role="presentation"> <a href="#cl-tab-section-3" id="cl-tabs-tab-3" role="tab" tabindex="-1">...</a> </li> ... </ul> <section id="cl-tab-section-1" role="tabpanel" tabindex="-1" aria-labelledby="cl-tabs-tab-1"> ... </section> <section id="cl-tab-section-2" role="tabpanel" tabindex="-1" aria-labelledby="cl-tabs-tab-2" hidden=""> ... </section> <section id="cl-tab-section-3" role="tabpanel" tabindex="-1" aria-labelledby="cl-tabs-tab-3" hidden=""> ... </section> ... </div>
